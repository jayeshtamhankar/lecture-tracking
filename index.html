Lecture Ledger

A running record of every class you teach — standard, division, group, topic, attendance, and status, all in one page.

01 Log a lecture

Standard, Division, and Group are required for every status except "No Class" — pick "No Class" and leave them blank to save as N/A. Topic, Learning Outcomes, and attendance are only required for a Regular Class. A Note is required when status is "No Class", to record the reason (meeting, function, holiday, etc.).

04 Monthly report

Generates a "Performance Report (Innovation Lab)" style PDF for one standard, division, and month — pulling straight from what you've already logged above. Each lecture in that month becomes one numbered session row.

05 Backup to Google Sheets

Push a copy of every lecture to a Google Sheet you own — set up once via a small Apps Script attached to your sheet (see data/google-apps-script.gs in this project for the code and setup steps). No Google Cloud Console needed. Photos aren't included (just a Yes/No flag) — they're too large for a spreadsheet cell.

Not connected

06 Local Excel backup

A second, no-account-needed way to keep a backup — downloads a real .xlsx file with your lectures and troublemaker notes as separate sheets. Keep copies wherever you like (the data/ folder in this project is a fine spot), and use Import if anything ever goes wrong.

07 Auto-save to a local data file

On Chrome or Edge on a computer, point this at your project's root folder (the one containing index.html) — every time you add, edit, or delete a lecture, data/lecture-ledger-data.json updates automatically, and any class photo you attach is also saved as a real image file inside photos/. This only works in Chrome/Edge on a desktop, and only when the site is opened over https (like your GitHub Pages link) — not on phones, Safari, Firefox, or opened directly as a local file. Treat it as a bonus layer alongside Google Sheets or Excel, not your only backup.
